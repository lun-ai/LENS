%%%%%%%%%%%%%%%%%%%% Summary %%%%%%%%%%%%%%%%%%%%
Here’s a simplified explanation of the four predicates for non-programmers:

1. **`isolated_behind`**:  
   Determines if one gate is "isolated behind" another. This is true if the gates are directly connected or if there’s a chain of connected gates between them, all marked as isolated. For example, if Gate A connects to Gate B, and both are isolated, then A is isolated behind B.

2. **`partition`**:  
   Splits a list of gates into two groups:  
   - The first group contains gates that are isolated behind a specific gate (and in the same circuit).  
   - The second group contains gates in the same circuit that are *not* isolated behind that gate.  
   This separation helps organize gates based on their isolation status.

3. **`partition_sizes`**:  
   Calculates the number of gates in each of the two groups created by `partition`. For instance, if the first group has 5 gates and the second has 3, it returns these counts as `[5, 3]`.

4. **`optimal_partition_sizes`**:  
   Evaluates all possible ways to split the gates using `partition_sizes` and selects the *most balanced* split. For example, between splits like `[5,3]` and `[4,4]`, it would choose `[4,4]` because the smaller group (4) is larger than in the first split (3), making it the better-balanced option.
